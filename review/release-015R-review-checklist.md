# Release-015R Review Checklist

## Pack 1 — Constitution and AI Governance

- [ ] Repository constitution is present.
- [ ] AI startup sequence is clear.
- [ ] Decision hierarchy is consistent with project practice.
- [ ] Bootstrap governance does not duplicate detailed governance documents.

## Pack 2 — Repository Governance

- [ ] Knowledge integrity is defined as a release gate.
- [ ] Repository health metrics are defined.
- [ ] Repository lifecycle is clear.
- [ ] Release governance requires repository-relative deltas.

## Pack 3 — Repository Standards

- [ ] Evidence priority standard is consistent with decision practice.
- [ ] Knowledge classifications are defined.
- [ ] Maintenance and delta standards are compatible with the repository.
- [ ] Authoring standard does not conflict with existing front matter.

## Pack 4 — QA and Roadmap

- [ ] Domain maturity levels are practical.
- [ ] Repository design principles match the existing architecture.
- [ ] Knowledge traceability rules are implementable.
- [ ] Integrity checklist can be used before every merge.
- [ ] Roadmap does not convert planned work into confirmed scope.

## Repository Safety

- [ ] Existing business knowledge is unchanged.
- [ ] No duplicate governance file was introduced.
- [ ] Navigation and cross-references remain valid.
- [ ] Any conflicts are documented for manual review.

## Final Decision

- [ ] Approved
- [ ] Approved with follow-up
- [ ] Rejected
