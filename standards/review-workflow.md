# Review Workflow

## Document Lifecycle

```text
Draft → Tony Review → Business Review → VR Review → Approved → Baseline
```

## Review Roles

| Role | Responsibility |
|---|---|
| Tony | BA knowledge verification |
| VR Business User | Business confirmation |
| Hopkins | Functional feasibility review |
| GLM | Document build / rendering |
| ChatGPT | BA authoring and knowledge maintenance |

## Review Output

All comments should be classified as:

- Correction
- Missing requirement
- Clarification
- Future enhancement
- Conflict
