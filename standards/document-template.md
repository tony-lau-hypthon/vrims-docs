# Document Template Standard

All VRIMS functional documents should follow this structure where applicable.

## Metadata

Each Markdown document should start with YAML front matter:

```yaml
---
id:
title:
type:
capability:
module:
version:
status:
owner:
reviewer:
knowledge_confidence:
source_sessions:
last_updated:
---
```
