# Repository Roadmap

## Purpose

Define the controlled evolution path of the VRIMS BA knowledge repository.

## Current State

- Domain knowledge is maintained as canonical current-state markdown.
- Releases are repository deltas.
- Git preserves history.
- AI models consume the repository through `AI-BOOTSTRAP.md`, `README.md`, and `INDEX.md`.

## Roadmap

### Phase 1 — Knowledge Consolidation
Status: In progress

- Consolidate workshop evidence.
- Establish module and capability boundaries.
- Separate confirmed facts, assumptions, conflicts, open questions, and decisions.

### Phase 2 — Repository Governance
Status: Release-015R

- Establish constitution and AI operating rules.
- Define evidence priority and knowledge classification.
- Introduce repository health and integrity review.
- Define traceability and maturity standards.

### Phase 3 — Knowledge Integrity Automation
Status: Planned

- Detect duplicate rules.
- Detect conflicting classifications.
- Detect broken links and orphan documents.
- Detect assumption leakage into confirmed knowledge.

### Phase 4 — Repository Health Automation
Status: Planned

- Generate health metrics.
- Track unresolved conflicts.
- Track pending VR decisions.
- Track document freshness and deprecated knowledge.

### Phase 5 — Decision Intelligence
Status: Planned

- Link decisions to evidence, modules, and impacted capabilities.
- Support change-impact analysis.
- Maintain supersession history.

### Phase 6 — Architecture Intelligence
Status: Planned

- Map business rules to capability ownership.
- Detect cross-module dependencies.
- Support functional-design and implementation traceability.

## Governance Rule

The roadmap may guide prioritisation but must not convert planned capabilities into confirmed requirements.
